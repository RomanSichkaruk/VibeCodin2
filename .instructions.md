# Claude Agent Customization Instructions

This file configures Claude's behavior in this workspace using MCP, subagents, and skills.

## Workspace Configuration

### Default Agent
When working in this repository, Claude should use the **code-review-agent** for all code-related tasks.

### File-Type Routing
```
*.js, *.ts, *.py, *.md  → code-review-agent
```

## MCP (Model Context Protocol)

### Git Server
- **Location**: `.claude/mcp/git.json`
- **Config**: `.claude/mcp/mcp.config.json`
- **Purpose**: Provide Git repository access for analysis

**Available Operations**:
- Get repository status
- Retrieve commit history
- Generate diffs between branches/commits

### Enabling MCP

To start using the Git MCP server:

```bash
# 1. Install MCP server (if needed)
npm install @modelcontextprotocol/server-git

# 2. Run the server
node mcp-servers/git-mcp.js

# 3. Claude will automatically connect via config in .claude/mcp/mcp.config.json
```

## Subagents

### Available Subagents

#### 1. Code Reviewer
- **File**: `.claude/subagents/code-reviewer.md`
- **Purpose**: Specialized code analysis and review
- **Triggers**: When reviewing PRs, commits, or code changes
- **Capabilities**:
  - Diff analysis
  - Code quality checks
  - Pattern detection
  - Best practice validation

**Example Invocation**:
```
"Use the code-reviewer subagent to analyze changes in the auth-refactor branch"
```

## Skills

### Available Skills

#### 1. Git Operations (`.claude/skills/git-operations.md`)
Utilities for git-related analysis:
- `analyze_git_diff()`: Compare branches/commits
- `get_commit_context()`: Retrieve commit metadata
- `get_branch_info()`: Understand branch state

#### 2. Code Quality (`.claude/skills/code-quality.md`)
Code analysis and validation:
- `check_code_standards()`: Validate code against standards
- `detect_patterns()`: Find anti-patterns and issues
- `suggest_improvements()`: Generate recommendations

### Using Skills

Skills are automatically available through the code-review-agent. They can also be used independently:

```
"Use the git-operations skill to analyze changes between main and feat/user-auth"
```

## Workflow Examples

### Review a Pull Request

```
User: "Review the changes in the feat/payment-integration branch"

Claude:
1. Uses git MCP to fetch diffs
2. Invokes code-reviewer subagent
3. Applies code-quality skill
4. Generates detailed review with issues and suggestions
```

### Analyze Recent Commits

```
User: "What's been committed to main in the last week?"

Claude:
1. Calls git MCP for commit history
2. Uses git-operations skill for context
3. Summarizes changes and authors
4. Highlights significant modifications
```

### Check Code Standards

```
User: "Do these changes follow our coding standards?"

Claude:
1. Retrieves code changes via git MCP
2. Applies check_code_standards() from code-quality skill
3. Reports violations and recommendations
4. Prioritizes by severity
```

## Adding Custom Components

### Create a New Skill

1. Create `.claude/skills/my-skill.md`
2. Define functions with parameters and return types
3. Add integration notes
4. Reference in agent file

### Create a New Subagent

1. Create `.claude/subagents/my-agent.md` with YAML frontmatter
2. List required skills and MCP servers
3. Define workflow and capabilities
4. Add to agent's subagent list

### Create a New MCP Server

1. Define configuration in `.claude/mcp/[server].json`
2. Register in `.claude/mcp/mcp.config.json`
3. Update agent to reference it

## Troubleshooting

### MCP Server Not Connected
- Check if Git MCP server is running
- Verify `.claude/mcp/mcp.config.json` has correct command path
- Check firewall/port restrictions

### Subagent Not Responding
- Verify `.claude/subagents/code-reviewer.md` exists
- Check YAML frontmatter is valid
- Ensure listed skills exist

### Skills Not Available
- Confirm skill files exist in `.claude/skills/`
- Check skill is registered in agent or subagent
- Verify function names in documentation
